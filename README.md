Проект: Тестирование функций на Python с использованием PyUnit

Описание проекта

Этот проект демонстрирует различные техники тестирования в Python с использованием встроенного модуля unittest. Проект включает три основные задачи, каждая из которых фокусируется на разных аспектах тестирования: базовое тестирование функций, параметрическое тестирование и тестирование обработки исключений.

Установка и настройка
Предварительные требования
Python 3.8 или выше

pip (менеджер пакетов Python)

Установка
Клонируйте репозиторий:

Создайте виртуальное окружение (рекомендуется):

bash

# Для Windows:
venv\Scripts\activate

Установите зависимости:

pip install -r requirements.txt

Задачи проекта:

Задача 1: Тестирование функции калькулятора

Цель: Создать и протестировать базовый калькулятор с четырьмя арифметическими операциями.

Функция: calculator(a, b, operation)
a, b - числа для операции
operation - строка с названием операции: 'add', 'subtract', 'multiply', 'divide'
Возвращает результат операции

Тесты проверяют:
Корректность сложения, вычитания, умножения и деления
Обработку целых и дробных чисел
Граничные случаи

Задача 2: Параметрические тесты для проверки четности чисел
Цель: Продемонстрировать использование параметрического тестирования.

Функция: is_even(num)
num - число для проверки
Возвращает True, если число четное, иначе False
Тесты проверяют:
Положительные, отрицательные числа и ноль
Большие числа
Использование subTest для параметрического тестирования

Задача 3: Тестирование обработки ошибок
Цель: Протестировать корректную обработку исключений.

Функция: safe_divide(a, b)
a, b - числа для деления
Возвращает результат деления a на b
Генерирует ZeroDivisionError при делении на ноль
Тесты проверяют:
Корректное деление при нормальных условиях
Генерацию исключения при делении на ноль
Использование assertRaises для проверки исключений

Запуск тестов:

Запуск всех тестов
python -m unittest discover tests

Запуск конкретного тестового файла
python -m unittest tests.test_calculator
python -m unittest tests.test_even_number
python -m unittest tests.test_division

Запуск с подробным выводом
python -m unittest discover tests -v

Принципы тестирования
В проекте применяются следующие принципы тестирования:
Изолированность тестов: Каждый тест независим от других
Читаемость: Тесты имеют понятные названия и структуру
Полнота покрытия: Тесты проверяют нормальные случаи, граничные условия и ошибки
Параметризация: Использование subTest для избежания дублирования кода
Тестирование исключений: Проверка корректной обработки ошибок

GitHub Actions
